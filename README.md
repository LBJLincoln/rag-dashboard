# RAG Dashboard

Static dashboard for monitoring the Multi-RAG Orchestrator project.

## Contents

- `docs/index.html` - Main dashboard HTML (live metrics, eval feed, trading board)
- `docs/status.json` - Current status data (auto-generated by mon-ipad)
- `docs/data.json` - Historical eval data (auto-generated by mon-ipad)
- `index.html` - Root dashboard with trading board and fallback support
- `control-panel.html` - Control panel for managing repos and operations
- `repos-config.json` - Configuration for the control panel
- `api/status.js` - API endpoint for live pipeline health (`/api/status`)
- `api/fallback-status.json` - Fallback data when backend unavailable

## Deployment

This repo is deployed to:
- Vercel: https://nomos-dashboard-alexis-morets-projects.vercel.app
- GitHub Pages (alternative)

## Data Updates

The JSON files are automatically updated by the mon-ipad control tower via:
```bash
python3 eval/generate_status.py  # generates status.json
# data.json is updated during eval runs
```

## Architecture

This is a **static dashboard** with serverless API endpoints for live data.

### Data Sources
- **External**: GitHub raw JSON (`status.json`, `data.json`)
- **API**: `/api/status` - Live pipeline health with trading board calculations
- **Fallback**: Local `fallback-status.json` when backend unavailable

### Features
- **Live Pipeline Monitoring**: Auto-refresh every 30 seconds
- **Trading Board**: BEST/WORST fixed positions + MIDDLE rolling placeholders
- **Fallback Mode**: Dashboard works offline with local data
- **API Endpoint**: `GET /api/status` returns structured pipeline health JSON

The dashboard reads from the n8n webhook endpoint:
- `https://lbjlincoln-nomos-rag-engine.hf.space/webhook/nomos-status`
